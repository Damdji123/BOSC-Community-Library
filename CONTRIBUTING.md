# Contributing to BOSC Community Library

Thank you for your interest in contributing to the BOSC Community Library! We welcome contributions from the community to help improve and expand this project.

## How to Contribute

### Reporting Issues
- Use the [issue templates](.github/ISSUE_TEMPLATE/) to report bugs or request features.
- Provide as much detail as possible, including steps to reproduce, expected behavior, and screenshots if applicable.

### Contributing Code
1. Fork the repository.
2. Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name`
3. Make your changes, ensuring they follow the project's coding standards.
4. Write or update tests if applicable.
5. Commit your changes with clear, descriptive commit messages.
6. Push your branch to your fork.
7. Create a pull request using the [pull request template](.github/PULL_REQUEST_TEMPLATE/).

### Development Setup
1. Clone the repository: `git clone https://github.com/Damdji123/BOSC-Community-Library.git`
2. Open `index.html` in your web browser to run the application.
3. For development, you can use any text editor or IDE that supports HTML, CSS, and JavaScript.

### Coding Standards
- Use consistent indentation (2 spaces for HTML/CSS, 2 spaces for JavaScript).
- Follow HTML5 and CSS3 best practices.
- Use meaningful variable and function names.
- Add comments for complex logic.
- Ensure the code works in modern browsers (Chrome, Firefox, Safari, Edge).

### Testing
- Test your changes in multiple browsers.
- Ensure that existing functionality is not broken.
- For new features, consider adding manual test cases.

### Commit Guidelines
- Use clear, descriptive commit messages.
- Start with a verb (e.g., "Add", "Fix", "Update").
- Keep commits focused on a single change.
- Reference issue numbers when applicable (e.g., "Fix #123").

### Pull Request Process
- Ensure your PR description clearly describes the changes and references any related issues.
- The PR will be reviewed by maintainers.
- Make any requested changes and push updates to your branch.
- Once approved, your PR will be merged.

## Code of Conduct
Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project, you agree to abide by its terms.

## Questions?
If you have any questions about contributing, feel free to open an issue or contact the maintainers.

Thank you for helping make the BOSC Community Library better!
